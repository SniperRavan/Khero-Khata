## ✦ What this project is

**Khero Khata** is a handcrafted productivity dashboard inspired by the **TaskFlow** concept screens, but adapted into your own darker, moodier, more cinematic interface. Instead of copying the light gray neomorphic mockups directly, this repo translates their layout logic, page system, and tactile UI philosophy into a custom **dark-mode soft UI** experience.

Right now, the project has the **Home page** implemented, and that page already establishes the product's identity: a left rail navigation, large greeting header, “My Day” tasks, “Upcoming Tasks,” a team activity feed, and a compact calendar block. The rest of the screens are planned using the future references you shared—**Projects, Team, Calendar, Settings, and Profile**—all staying inside the same TaskFlow-inspired product vision.

***

## ✦ Inspiration vs identity

This repo should be understood like this:

- **TaskFlow** = the design inspiration and page blueprint.
- **Khero Khata** = your actual product name, repo identity, and implementation.
- **Current implementation** = dark neomorphic Home page, already customized beyond the original mockup direction.
- **Future build** = full multi-page productivity suite based on the UI language shown in the provided reference screens.

So the README should not pretend everything is finished yet—it should present the project as a **high-quality evolving frontend system** with a clear visual roadmap.

## ✦ Design system direction

The original reference set describes a soft neomorphic system built around light gray surfaces, diffused highlights, inset controls, and a restrained purple accent. Your implementation already shifts that into a darker interpretation while preserving the same physical metaphor: surfaces feel **raised**, **pressed**, **carved**, or **glowing softly inside the panel**.

### Core visual ideas

- **Base personality** → soft, tactile, calm
- **Main metaphor** → cards feel physically formed from the UI surface
- **Accent behavior** → purple used for emphasis, not overload
- **Spacing** → roomy, premium, editorial
- **Typography** → modern sans-serif with strong dashboard hierarchy
- **Interaction feel** → press, inset, glow, elevation change

## ✦ Project structure

```text
Khero-Khata/
├── index.html              # SPA shell — navigation sidebar & view container
├── README.md               # Project documentation
├── .gitignore              # Git ignore rules
├── assets/
│   └── icons/              # SVG / icon assets
├── styles/
│   ├── base.css            # Resets, typography & structural layout
│   ├── components.css      # Cards, buttons, inputs & neomorphic UI physics
│   └── themes.css          # CSS Variables — colors, shadows & spacing tokens
└── js/
    ├── router.js           # SPA navigation & HTML template injection
    ├── logic.js            # Interactive DOM logic (tasks, checkboxes)
    └── theme.js            # Runtime theme switching logic
```

### Structure notes

- `index.html` → main shell of the experience.
- `styles/base.css` → layout, reset, typography, foundations.
- `styles/components.css` → cards, widgets, buttons, panels, task blocks.
- `styles/themes.css` → visual tokens, colors, shadows, theme behavior.
- `js/router.js` → navigation and view handling.
- `js/logic.js` → task interactions and dynamic behavior.
- `js/theme.js` → theme-related control and future expansion.

## ✦ Getting started

### Clone the repository

```bash
git clone https://github.com/SniperRavan/Khero-Khata.git
cd Khero-Khata
```

### Run locally

For a quick preview, open `index.html` in your browser.

For a smoother development workflow, use **VS Code + Live Server**:

1. Open the folder in VS Code.
2. Install the **Live Server** extension.
3. Right-click `index.html`.
4. Choose **Open with Live Server**.

This is the better workflow if your JavaScript modules or routing behavior expect a local server environment.
